_parse_message(self, string, protocol)

source code 

Parse the contents of a message. Also do processing outside the reactor thread.

This method should run in a separate thread to prevent blocking the reactor.

Both the parsed original message and a possible return message are returned in a special MessageResult instance.

_r_process_message(self, result, protocol)

source code 

Perform processing required inside the reactor thread. Expects the original message to be already parsed. Possibly a return message is already present from actions processed outside the reactor.

Returns the message to be send back to the client.

_r_send_result(self, response, protocol)

source code 

Send the result back to the client. Response is expected to be a string ready to send.

_r_send_error(self, result, protocol)

source code 

Send an exception to the client. This is the only method that creates XML inside the reactor thread. This might cause problems if the error contains a lot of data, but for now we do not expect this to be the case.
